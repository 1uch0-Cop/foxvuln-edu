# Contribuir

Las contribuciones deben conservar el propósito educativo y defensivo del proyecto.

No se aceptarán funciones de explotación automática, fuerza bruta, evasión, denegación de servicio, escaneo masivo, objetivos públicos o mecanismos para omitir la allowlist.

Antes de proponer cambios:

```bash
./verify_release.sh
```

Incluya pruebas para cualquier modificación de política, perfiles, tráfico o manejo de evidencias.
